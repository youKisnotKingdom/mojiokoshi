# 文字起こし・サマライズ ウェブアプリ ロードマップ

## 概要

音声ファイルをアップロードし、オンプレGPU環境でWhisper/qwen-asrを用いて文字起こしを行い、外部LLM API経由でサマライズを生成するウェブアプリケーション。

## 技術スタック

| 項目 | 技術 |
|------|------|
| フロントエンド | HTMX + Jinja2テンプレート |
| バックエンド | FastAPI (Python) |
| データベース | PostgreSQL |
| 文字起こし | Whisper / qwen-asr (16GB GPU) |
| サマライズ | 外部LLM API (OpenAI互換) |
| デプロイ | Docker Compose |

## 機能一覧

- **文字起こし画面**: 音声ファイルアップロード、文字起こし実行
- **一覧画面**: 音声データ、文字起こし生データ、サマライズの閲覧・管理
- **ユーザー管理**: 管理者 + 一般ユーザー（6桁ID + パスワード認証）
- **自動廃棄**: 音声ファイルの自動削除（期間設定可能、デフォルト30日）

## フェーズ構成

### Phase 1: プロジェクト基盤構築
**目標**: 開発環境とプロジェクト構造の整備

- FastAPIプロジェクトの初期化
- ディレクトリ構造の設計
- 依存関係の定義（requirements.txt / pyproject.toml）
- 基本的なHTMXテンプレート構造
- 開発用Docker Compose設定
- PostgreSQL接続設定

### Phase 2: 認証・ユーザー管理
**目標**: ログイン機能と権限管理の実装

- ユーザーモデル設計（6桁ID、パスワードハッシュ、ロール）
- 管理者・一般ユーザーのロール定義
- ログイン/ログアウト機能
- セッション管理
- 管理者用ユーザー管理画面（CRUD）
- 初期管理者作成スクリプト

### Phase 3: 文字起こし機能
**目標**: 音声ファイルのアップロードと文字起こし処理

- 音声ファイルアップロード機能
- ファイル保存・管理（ストレージ設計）
- Whisper / qwen-asr の統合
- 非同期ジョブキュー（バックグラウンド処理）
- 文字起こし進捗表示
- 複数フォーマット対応（ffmpegによる変換）

### Phase 4: サマライズ機能
**目標**: 文字起こし結果のLLMサマライズ

- 外部LLM API連携（OpenAI互換）
- APIエンドポイント設定（設定可能）
- サマライズ実行機能
- プロンプト設定・カスタマイズ
- サマライズ結果の保存

### Phase 5: 一覧・管理画面
**目標**: データの閲覧・管理インターフェース

- 文字起こしジョブ一覧画面
- 詳細画面（音声再生、文字起こし表示、サマライズ表示）
- 検索・フィルタ機能
- ページネーション
- データエクスポート機能

### Phase 6: 自動廃棄機能
**目標**: 音声ファイルの自動削除機能

- 廃棄設定モデル（期間設定）
- 管理画面での設定変更
- 定期実行ジョブ（スケジューラ）
- 廃棄ログ・通知

### Phase 7: Docker化・デプロイ
**目標**: 本番環境向けのコンテナ化

- 本番用Dockerfile作成
- Docker Compose構成（FastAPI, PostgreSQL, Worker）
- GPU対応設定（NVIDIA Container Toolkit）
- 環境変数・シークレット管理
- Nginx/リバースプロキシ設定
- ヘルスチェック・監視設定

### Phase 8: テスト・ドキュメント
**目標**: 品質保証とドキュメント整備

- ユニットテスト
- 統合テスト
- E2Eテスト
- API仕様書（OpenAPI）
- ユーザーマニュアル
- 運用マニュアル
